◆ Timer_A()

__interrupt void Timer_A ( void  )

instrukcje wykonywane przy przerwaniach spowodowanych TimerA

wpierw sprawdzenie catchStatus dla obu rzędów, następnie ustawienie odpowiedniego znaku pola łapania(z kafelkiem lub bez),

następnie przesunięcie wszystkich pozostałych kafelków w tablicy, generowanie nowych i wywolanie moveTiles(),

na koniec przywrócenie BUTTONxUNUSED do wartości 1 i obsługa sytuacji kończących grę

przy lives==0 lub catchStatus[x]==2 kończy grę resetując stan zmiennych i wywołując endOfGame()

Dokumentacja zmiennych

◆ BUTTON1UNUSED

int BUTTON1UNUSED = 1

powstrzymuje problemy z "przytrzymywaniem" pierwszego przycisku, zezwala na jedno wciśnięcie owego przycisku na interwał

1 oznacza, że można wcisnąć, 0 blokuje wciskanie

◆ BUTTON2UNUSED

int BUTTON2UNUSED = 1

powstrzymuje problemy z "przytrzymywaniem" drugiego przycisku, zezwala na jedno wciśnięcie owego przycisku na interwał

1 oznacza, że można wcisnąć, 0 blokuje wciskanie

◆ catchStatus

int catchStatus[2] = {0, 0}

zmienna jest tablicą przechowującą status obu rzędów LCD

wynosi 0, gdy kafelka nie trzeba łapać lub wprost po złapaniu

wynosi 1, gdy kafelek należy teraz złapać

wynosi 2, gdy kafelka nie złapie się w porę, co powoduje przegraną

◆ gameStatus

int gameStatus = 0

zmienna przechowuje informację, czy gra aktualnie trwa, czy nie

wynosi 0, gdy gra nie jest w toku (menu, tabela wyników, odliczanie, ekran porażki)

wynosi 1, gdy gra jest w toku
